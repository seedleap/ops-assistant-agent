# Creator Support v7 时区时间与记忆收窄迭代

日期：2026-07-24

## 目标

在不增加 revision 4291 一期场景的前提下，优化 System Prompt、Skill 分工、工具名称和输入输出格式，并把原始消息恢复升级为轻量分层记忆。

## 基线与变量

- 基线：commit `197b1ea`。
- 候选：v7 Prompt/Skill/Tool/Memory。
- 保留：公开作品范围、上下文 UID 绑定、PID/URL 解析、Creator IM 与 Outreach 隔离。
- 撤除：业务结果缺少 `ok + as_of` 时的运行时强制失败。

## 假设

1. System Prompt 只保留权威顺序、路由、统一工具协议、记忆和安全边界，具体分析方法下沉 Skill，可减少重复和指令冲突。
2. 按作品、评论、账号、文档拆成四个垂类 Skill，比一个综合分析 Skill 更容易触发正确方法。
3. 按业务对象命名的 `query_public_work / analyze_work_comments / query_creator_account_summary` 能让工具边界与 Skill 路由保持一致。
4. `project_ref / detail_level` 输入和 `data / meta / error` 输出比旧的混合命名更稳定。
5. 长期记忆只保留用户明确偏好和客户端提供的 IANA 时区；当前时间每轮动态生成，不作为历史事实长期累积。

## 非目标

- 不增加灵感、Prompt 优化、可视化、发布器或活动写操作。
- 不使用模型生成长期画像。
- 不要求上游 MCP 立刻完整实现 `meta` 字段；适配器只做兼容归一化，不强制失败。
- 不把本地测试等同于真实模型路由效果。

## 本地评测集

- 现有 revision 4291 路由用例；
- 四个 Skill/Profile 映射；
- 工具新旧参数映射与统一输出；
- 缺少 `as_of` 时仍可返回部分数据；
- 明确偏好和 IANA 时区的跨线程恢复；
- 每轮 UTC 时间、本地时间、采用时区和时区来源注入；
- Pi 内置 `read` 与自定义工具组合；
- 全量类型检查、单测、构建和 Skill 校验。

## 接受标准

- Creator IM 保持三个数据工具和四个垂类 Skill；
- Outreach 只保留一个活动参与工具；
- 所有业务工具对 Agent 返回 `data / meta / error`；
- 记忆可跨线程恢复偏好和时区，但不保存作品引用；
- 原有回归无新增失败。

## 已知不确定项与暂不实现

- 客户端是否已经能稳定提供 IANA 时区尚未确认。接口现已接受可选 `timezone`；未传时沿用已存时区，再没有则使用 `UTC`。
- 暂不通过 IP、语言、国家字段或聊天文本推断时区，避免把猜测写成用户事实。
- “时区时间”拆为两类：时区可持久化；当前时间每轮动态生成。当前时间不写入长期记忆，避免迅速过期。
- 按本轮要求，暂不实现敏感信息/提示词注入过滤，也不实现自然语言“忘掉记忆”的删除动作。偏好抽取仍是简单规则，不代表完整隐私治理。
- 偏好抽取以固定中英文触发词和句号边界为准，可能漏记、截断或误记；暂不引入模型抽取与置信度体系。
- 暂不提供独立的记忆管理 API 或设置页；若后续需要真实删除、查看或修正记忆，应单独定义鉴权、审计和产品交互。
