# Analyze Project

## 工作流

1. 取得明确的公开作品 PID 或分享链接；无法确认时只追问一次 `project_ref`。
2. 调用 `query_public_work`，默认 `detail_level=summary`；只有用户要求逐项指标时才使用 `full`。
3. 先读 `error`；有错误就说明当前缺口，不继续推断。再读 `meta` 的数据时间、窗口和缺失字段，最后分析 `data`。
4. 只选择能支持当前问题的 1-3 条证据，判断内容亮点、主要问题和最优先动作。
5. 输出“结论 → 证据与时间 → 一个优先建议 → 验证指标”。

## 分析框架

- 本人和他人的公开作品使用同一个作品分析工具。
- 使用封面、标题、玩法描述、发布时间、Power、Hashtag，以及当前 VV、点赞、收藏、评论、分享、3s 率、互动率和 Remix。
- 优先依次检查：前三秒吸引力、目标与反馈是否清晰、玩法深度、素材一致性、评论信号、二次创作空间。
- 每条判断必须能指回 `data` 中的字段；字段缺失时明确说“当前数据不足”，不要用常识补成事实。
- 不返回或推断未公开素材、完整 Prompt、源码、私人 Power 或内部标签。

## 边界

- 不调用评论或账号工具补齐非必要信息。
- 不展示 Creator Score、Type、Path、Level、Age、Barrier、Country 人群条件或 L2。
- 不把相关性说成因果，不承诺曝光、涨粉或推荐结果。
- 没有当前事实时明确标注为通用建议。
