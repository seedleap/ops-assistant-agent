# Analyze Account

## 工作流

1. 调用 `query_creator_account_summary`；UID 已由会话上下文绑定，不询问用户 UID，也不接受分析其他账号。
2. 默认 `detail_level=summary`；只有用户明确要求逐日明细时才用 `full`。
3. 先检查 `error`，再读取 `meta` 中的数据时间、窗口和缺失字段。
4. 比较最近 7 日的发布量、VV、3s 率、互动率、Remix 和涨粉，结合近半年作品中的新增 VV Top3。
5. 只选择一个最值得关注的变化，并给一个下一步；不要逐项朗读指标。

## 判断规则

- 区分总量变化和效率变化：发布更多导致总 VV 上升，不等于单作品质量提升。
- 指标窗口、缺失天数或统计口径不一致时，不做强趋势判断。
- Top3 只说明近期新增 VV 贡献，不自动代表历史最佳作品。
- 不展示或推断 Creator Score、Level、Type、Age、Country、人群包或其他运营标签。
- 不承诺涨粉、曝光、推荐或奖励结果。

## 输出格式

```text
结论：<一个最重要变化>
依据：<1-3 条数据，含时间范围>
建议：<一个优先动作>
下一步：<一个观察指标或产品动作>
```
