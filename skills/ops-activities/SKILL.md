# Ops Activities

## 使用步骤

1. 先使用 Pi 内置 `read` 读取本 Skill 目录下相关的 `docs/*.md`，确认活动名称、入口、玩法和适用人群。
2. 如果问题涉及资格、报名、任务进度、奖励、频控或活动卡片，必须调用 `query_creator_activity_status`；知识库只能说明规则，不能证明用户状态。
3. 结合创作者已确认的年龄路线、创作方向和作品事实说明“为什么可能适合”。
4. 缺少资格确认、有效入口或活动时效信息时，不发送确定性邀请，只说明缺少的信息或建议用户以官方活动页为准。

## 事实边界

- 不从活动文档推断用户已获得资格、已报名、已完成任务或已到账奖励。
- 不自行计算任务进度、修改活动规则、代替报名、领取奖励或更新卡片状态。
- 不向用户展示 Creator Score、Level、Age 或其他圈选标签；只解释公开的参与条件与已确认结果。
- 不承诺曝光、流量、排名、升级、积分或其他结果；除非上游系统明确返回对应状态。
- 活动链接、截止时间和权益以当前知识库或活动中台返回为准；不确定时明确标注。
- 运营标签只是候选条件，不等同于活动资格。

## 输出要求

- 推荐时说明：活动是什么、为什么可能适合、用户下一步做什么。
- 一次只推荐一个最相关的活动或动作，避免同时推广多个目标。
- 使用简洁、友好、无压力的表达，不制造焦虑、攀比、虚假稀缺或金钱诱导。
- 对可能未满 13 岁的用户，只推荐当前年龄路线明确允许的活动和任务。
- 如果用于主动触达，只有在资格、活动有效性和官方入口都确认后才生成可发送文案；否则返回 `NO_OUTREACH` 及简短原因。
