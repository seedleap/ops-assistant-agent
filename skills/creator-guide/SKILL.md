# Creator Guide

## 适用场景

当用户询问以下内容时使用本 Skill：

- Loopit 能做什么、适合做哪些类型的作品；
- 如何设计玩法、提示词、画面风格或首屏体验；
- 如何根据通用方法改进作品；
- 想参考热门案例或理解案例的可迁移做法。

## 使用步骤

1. 先判断用户要的是平台能力说明、通用创作建议，还是案例拆解。
2. 使用 Pi 内置 `read` 读取本 Skill 目录下与问题相关的 `docs/*.md`，不要只依据本文件的目录描述回答细节。
3. 如果用户提到具体作品、数据、评论或创作过程，优先使用对应的业务查询工具获取当前事实，再结合本 Skill 给建议。
4. 输出“建议做什么、为什么、如何验证”，一次聚焦一到三个最重要的动作。

## 参考文档

使用 `read` 读取以下真实文件，不要猜测文件名：

- `docs/loopit-capabilities.md`：Loopit 能力范围、适合的作品类型和产品限制。
- `docs/hot-case-studies.md`：热门案例、玩法机制和可迁移的创作方法。

## 内容边界

- 本 Skill 提供方法和参考，不证明某个作品当前的排名、数据、资格或官方推荐状态。
- 案例中的数据、链接和结论可能随运营更新；引用前以知识库当前文档为准，不把案例结果承诺给用户。
- 不把案例中的相关性说成确定因果，也不声称某个 Prompt 或玩法必然上热门。
- 不编造 Loopit 尚未确认支持的能力；不确定时明确说明需要以产品当前能力为准。
- 对可能未满 13 岁的用户，建议保持安全、简单，不引导收集个人信息或转移到外部私聊。

## 输出要求

- 使用用户当前语言，表达简洁、具体、可执行。
- 没有具体作品时，明确标注这是通用建议。
- 有具体作品时，优先引用已查询到的事实，并给出一个可验证的下一步。
- 避免“增加趣味性”“提升体验”这类没有行动定义的空泛表达。
