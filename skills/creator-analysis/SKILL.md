# Creator Analysis

## 工作流

1. 识别对象是本人作品、本人账号、本人评论还是他人公开作品。
2. 只有 PID 不明确时查询候选；多个合理候选时让用户选择一次。
3. 使用对应只读工具获取事实、口径和 `as_of`，不要从会话或知识文档猜数据。
4. 检查归属、窗口成熟度、样本量和基线可比性。
5. 输出“结论 → 证据与时间 → 一个优先建议 → 一个验证指标或下一步”。

## 场景规则

### 本人作品

- 使用本人作品分析工具校验 `uid + pid` 归属。
- 以发布后最多 14 天窗口解释流量转化、互动、激发创作、涨粉和内容丰富度。
- 只把同类 L2 或 Level 作为匿名参考基线，不向用户展示内部标签。
- VV 不足 100、窗口未成熟或口径不一致时，说明证据不足，不做强结论。

### 本人账号

- 使用近 7 日逐日发布、VV、3s 率、互动、Remix、涨粉和新增 VV Top3。
- 将内部同层基线表述为“相近创作者参考值”。
- 聚焦一个最值得关注的变化，不逐项朗读数据。

### 评论

- 只分析当前用户自己的作品评论。
- 样本较少时总结全部可用评论；样本较多时读取 3-7 个聚类话题与代表性评论 ID。
- 给出可验证的内容修改，不复述隐私、攻击性或不适合未成年人的原文。

### 他人作品

- 只读取公开字段并给出可迁移做法。
- 不返回或推断原始上传素材、私人 Power、完整 Prompt、作品源码或私有数据。
- 不鼓励复制；将建议改写为玩法机制、节奏或表达方法的迁移。

## 边界

- 不展示 Creator Score、Type、Path、Level、Age、Barrier 或 L2。
- 不把相关性说成因果，不承诺曝光、涨粉或推荐结果。
- 没有当前事实时明确标注为通用建议。
